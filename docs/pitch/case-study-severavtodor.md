| **Перерасход бюджета** | +25% к плановой смете | 120+ млн ₽/год |
| **Срыв сроков** | +25% к плановому периоду | Штрафы, простой техники |
| **Информационный хаос** | Данные в Excel, почте, мессенджерах | Потери 50+ млн ₽/год |
| **Ручной контроль** | Руководитель тратит 60% времени на статус-обновления | Потеря управленческого ресурса |
| **Риск-менеджмент** | Реактивный, а не проактивный | 15-30 млн ₽/год на устранение |
| **Разрозненность** | 7+ проектов без единой системы координации | Дублирование ресурсов |

### Конкретные примеры:

1. **Проект А:** перерасход 32% из-за неучтённых логистических затрат (доставка зимой)
2. **Проект Б:** срыв сроков на 4 месяца — несогласованные поставки с двумя субподрядчиками
3. **Проект В:** потеря 8 млн ₽ на переделках из-за ошибки в смете (не обнаружена вовремя)

### Количественная оценка потерь

## 3. Решение

### Этап 1: КСУП (Корпоративная Система Управления Проектами)

**Основание:** MBA дипломная работа, РАНХиГС, 2025.

```
MBA диплом → Методология КСУП → Адаптация под «Северавтодор» → Внедрение
  (2025)        (июнь 2025)         (июль-август 2025)        (сент 2025)
```

**Что включала КСУП:**
- Регламент управления проектами (под строительство ЯНАО)
- Шаблоны документов: 40 документов, 580+ страниц
- Роли и ответственности (RACI-матрица)
- Процессы: инициация → планирование → контроль → закрытие
- Система показателей (KPI) для каждого проекта
- Процедура риск-менеджмента

**Масштаб:** ~1700 тестовых сценариев для проверки процессов.

### Этап 2: AI-PMO (AI-усиление КСУП)

**Основание:** практическая необходимость автоматизации ручных процессов КСУП.

```
КСУП внедрена → Выявлены ручные процессы → AI-автоматизация → CEOClaw
  (сен 2025)     (октябрь 2025)           (ноябрь 2025)      (2026)
```

**Что автоматизировано:**
- Сбор статусов с проектов → AI-агент парсит отчёты
- Прогнозирование рисков → ML-модель на исторических данных
- Генерация сводных отчётов для CEO → 1 клик вместо 4 часов
- Мониторинг отклонений бюджета/сроков → автоматические уведомления
- Рекомендации по оптимизации ресурсов → AI-анализ загрузки

### Этап 3: CEOClaw (продуктизация)

**Переход от внутреннего инструмента к SaaS-продукту:**

- Время от MBA диплома до продукта: **<6 месяцев**
- Проверка на реальных проектах: **7+ simultaneously**
- Код написан и поддерживается: AI-ассистентами (OpenClaw)
