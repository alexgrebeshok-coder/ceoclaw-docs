# Contributing to CEOClaw Documentation

Thank you for your interest in contributing to CEOClaw's documentation!

## How to Contribute

### Reporting Issues
- Found a typo? [Open a bug report](../../issues/new?template=bug_report.md)
- Want new content? [Open a content request](../../issues/new?template=feature_request.md)
- Have a question? [Ask here](../../issues/new?template=question.md)

### Submitting Changes

1. **Fork** this repository
2. **Create a branch:** `git checkout -b my-improvement`
3. **Make changes** following our guidelines below
4. **Commit:** `git commit -m "Add: description of change"`
5. **Push:** `git push origin my-improvement`
6. **Open a Pull Request**

### Guidelines

- **Language:** English for documentation (Russian content goes in `/docs/press/` and `/blog/`)
- **Tone:** Professional but accessible — construction PMs, not software engineers
- **Accuracy:** All metrics and claims must be verifiable
- **No code:** This repository contains documentation only — no source code
- **No secrets:** Never commit API keys, credentials, or private data
- **Images:** Optimize screenshots (PNG, <500KB each), use WebP for photos

### Content Structure

```
docs/
├── getting-started.md    # What is CEOClaw, who is it for
├── features.md           # Feature list
├── architecture.md       # High-level tech architecture
├── roadmap.md            # Development timeline
├── pricing.md            # Pricing philosophy
├── use-cases/            # Industry-specific examples
├── ai-pmo/               # AI-PMO concept and agents
├── investors/            # Investor-focused content
├── partners/             # Partnership information
└── press/                # Press kit materials
```
